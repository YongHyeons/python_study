# 뉴스 수집 및 키워드 분석기 — 프로젝트 가이드라인

---

## 1. 프로젝트 개요

이 프로젝트는 **Python**을 활용하여 뉴스 데이터를 수집하고, DB에 저장한 뒤,
키워드 검색과 키워드 빈도 분석을 수행하는 **학습용 1인 프로젝트**이다.

### 최종 흐름

```
RSS 또는 뉴스 페이지에서 기사 목록 수집
→ 기사 제목 / 링크 / 날짜 추출
→ 기사 본문 수집
→ DB에 저장
→ 키워드 검색
→ 키워드 빈도 분석
→ CSV 또는 HTML 리포트 생성
```

---

## 6. DB 테이블 설계

처음에는 `news` 테이블 하나만 사용한다.

### news 테이블

| 컬럼명 | 타입 | 설명 |
|--------|------|------|
| id | INTEGER PRIMARY KEY AUTOINCREMENT | 고유 번호 |
| title | TEXT NOT NULL | 기사 제목 |
| url | TEXT UNIQUE NOT NULL | 기사 링크 (중복 방지) |
| source | TEXT | 언론사 또는 출처 |
| published_at | TEXT | 발행일 |
| content | TEXT | 기사 본문 |
| collected_at | TEXT | 수집한 날짜 |

### SQL 생성문

```sql
CREATE TABLE IF NOT EXISTS news (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    title TEXT NOT NULL,
    url TEXT UNIQUE NOT NULL,
    source TEXT,
    published_at TEXT,
    content TEXT,
    collected_at TEXT
);
```

> **중요**: `url`은 `UNIQUE`로 관리한다.
> 같은 뉴스가 여러 번 저장되지 않도록 하기 위함이다.

## 7. config.xml 설계

XML 학습을 위해 설정 파일을 XML로 만든다.

### 포함할 정보

```
- RSS 주소 목록
- 분석할 키워드 목록
- 수집 개수 제한
```

### config.xml 예시

```xml
<?xml version="1.0" encoding="UTF-8"?>
<config>
    <rssFeeds>
        <feed name="example1">https://example.com/rss</feed>
        <feed name="example2">https://example2.com/rss</feed>
    </rssFeeds>

    <keywords>
        <keyword>파이썬</keyword>
        <keyword>AI</keyword>
        <keyword>데이터</keyword>
    </keywords>

    <limit>10</limit>
</config>
```

### 이 단계에서 학습할 내용

```
- XML 태그 구조
- xml.etree.ElementTree 사용법
- 태그 찾기 (find, findall)
- 텍스트 값 가져오기 (.text)
- 속성 값 가져오기 (.attrib)
- 반복문으로 여러 feed 읽기
```

## 8. RSS XML 구조 참고

RSS는 일반적으로 아래와 같은 XML 구조를 가진다.

```xml
<rss>
  <channel>
    <title>뉴스 채널 이름</title>
    <item>
      <title>기사 제목</title>
      <link>기사 링크</link>
      <pubDate>발행일</pubDate>
    </item>
    <item>
      <title>다른 기사 제목</title>
      <link>다른 기사 링크</link>
      <pubDate>발행일</pubDate>
    </item>
  </channel>
</rss>
```

### Milestone 1: RSS 읽기

**목표**: RSS URL 1개에서 기사 제목, 링크, 날짜를 가져와 화면에 출력한다.

**해야 할 일**:
1. RSS URL 정하기
2. RSS URL에 HTTP 요청 보내기
3. XML 문자열 받기
4. XML 파싱하기
5. `item` 태그 반복하기
6. `title`, `link`, `pubDate` 추출하기
7. 화면에 출력하기

**예상 출력**:
```
[1] 뉴스 제목 A
    https://example.com/news/a
    2026-06-09

[2] 뉴스 제목 B
    https://example.com/news/b
    2026-06-09
```

**완료 기준**: 뉴스 10개 정도가 정상 출력된다.

**해야 할 일**:

### Milestone 6: 키워드 검색

**목표**: 저장된 뉴스 제목에서 키워드 검색이 가능하게 한다.

**해야 할 일**:
1. 사용자에게 검색어 입력받기 → `input()`
2. title에서 키워드 검색하기 → `select_news_by_keyword(keyword)`

**SQL 개념**:

**출력 예시**:
```

[검색 결과]

```

**완료 기준**: 사용자가 입력한 키워드가 포함된 기사만 출력된다.

### Milestone 7: 기사 본문 크롤링

**목표**: 기사 링크에 접속해서 본문을 가져오고 DB에 저장한다.

**해야 할 일**:
1. 기사 URL에 HTTP 요청 보내기
2. HTML 받아오기
3. BeautifulSoup으로 파싱
4. 본문 영역 찾기
5. 본문 텍스트 추출
6. DB의 `content` 컬럼 업데이트

**BeautifulSoup 학습 포인트**:
```
- find()           : 태그 하나 찾기
- find_all()       : 태그 여러 개 찾기
- select()         : CSS 선택자로 여러 개 찾기
- select_one()     : CSS 선택자로 하나 찾기
- get_text()       : 텍스트만 추출
- attrs            : 속성 가져오기
- class / id 선택자
```

**본문 추출 시 고려할 것**:
```
- 광고 제거
- 기자명 제거
- 관련기사 제거
- 불필요한 공백 제거
- 줄바꿈 정리
```

**주의사항**:
> 뉴스 사이트마다 HTML 구조가 다르다.
> 처음에는 **사이트 하나만** 정해서 본문 추출 규칙을 만든다.
> 여러 사이트를 한 번에 처리하려 하면 난이도가 급격히 올라간다.

**완료 기준**: `content` 컬럼에 기사 본문이 저장된다.

### Milestone 13: Selenium 추가 (후반부)

> Selenium은 초반부터 사용하지 않는다.
> RSS, BeautifulSoup, DB 저장이 익숙해진 뒤에 추가한다.

**Selenium이 필요한 경우**:
```
- 페이지가 JavaScript로 나중에 로딩되는 경우
- 더보기 버튼을 눌러야 하는 경우
- 무한 스크롤이 필요한 경우
- 검색창에 입력하고 버튼을 눌러야 하는 경우
- 로그인이 필요한 경우
```

**Selenium으로 추가할 기능**:
1. 뉴스 검색 페이지 열기
2. 검색어 입력
3. 검색 버튼 클릭
4. 결과 페이지 스크롤
5. 기사 링크 목록 수집
6. 수집한 링크를 DB에 저장

**완료 기준**: 브라우저가 자동으로 실행되고 뉴스 링크를 수집한다.

## 10. DB 함수 설계 가이드

`db.py`에서 만들어야 할 함수 목록:

| 함수명 | 역할 |
|--------|------|
| `connect_db()` | DB 연결 |
| `create_table()` | news 테이블 생성 |
| `insert_news(news)` | 뉴스 한 건 저장 |
| `select_all_news()` | 전체 뉴스 조회 |
| `select_news_by_keyword(keyword)` | 키워드로 검색 |

| `get_news_without_content()` | 본문이 없는 뉴스 조회 |

## 11. 예외 처리 계획

프로젝트가 어느 정도 동작하면 예외 처리를 추가한다.

### 처리해야 할 상황

| 상황 | 대응 |
|------|------|
| 인터넷 연결 실패 | 오류 메시지 출력, 재시도 또는 건너뛰기 |
| RSS 주소 오류 | 해당 RSS 건너뛰고 다음 RSS 처리 |
| XML 파싱 실패 | 오류 로그 출력 |
| 기사 링크 접속 실패 | 해당 기사 건너뛰기 |
| 본문 영역을 찾지 못함 | content를 빈 문자열로 처리 |
| DB 중복 저장 | UNIQUE 제약조건으로 자동 방지 |
| DB 연결 실패 | 오류 메시지 출력 후 종료 |
| 빈 검색어 입력 | 다시 입력 요청 |

**목표**: 오류가 발생해도 프로그램이 바로 종료되지 않게 한다.
